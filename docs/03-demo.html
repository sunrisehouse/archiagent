작성·적재 두 경로 — 모든 데모의 토대

💬 "인터뷰로 요구사항부터 작성하자" · "기존 아키텍처 설계서 적재해줘" (+ 문서)

영상에서 보여줄 것

  • ① 작성 — 신규 산출물을 인터뷰로 만들며 그래프 적재 (Demo 1 상세)
  • ② 적재 — 기존 문서를 kb_artifact_ingest로 LLM 추출 → Artifact 노드 + 추적성 엣지
  • 어느 경로든 요구→설계→테스트 추적성으로 연결 — 같은 그래프

문서 작성 — 분석 → 설계, 자연어로

💬 "고객 인터뷰로 요구사항 정의서 잡고, ToBe는 AsIs처럼 노드 대수 유지해서 설계서까지"

영상에서 보여줄 것

  • 인터뷰 메모 → 요구사항 정의서(R-ID) 초안, 승인 시 그래프 적재
  • 현행 분석서 → As-Is 노드 대수·용량이 그래프 사실로
  • "AsIs처럼 노드 대수 유지" 지시 → 에이전트가 kb_graph_query로 제약 읽어 ToBe 설계서 초안 · 충돌 경고 · DERIVED_FROM 추적성 자동
