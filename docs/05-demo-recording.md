# 산출물 부록 — 시연 영상 녹화 가이드 (Mac · 실제 모델)

> 은행 차세대 시나리오(5.1~5.4)를 실제 에이전트가 실제 모델(Claude Max 구독)로 처리하는 모습을 녹화한다.
> 재생은 `scripts/demo.py`(데모 플레이어)가 대신하고, **당신은 준비 → 실행 → 화면 녹화 → 편집**만 하면 된다.
> 시나리오 원문은 `04-scenario-walkthrough.md`, 슬라이드는 `01`·`02`·`03`.

---

## 0. 한눈에 — 내가 할 일 체크리스트

- [ ] **준비**: 코드 최신화 + 설치 + `claude` CLI 로그인 확인 (§1)
- [ ] **리허설**: `--fake`로 한 번 돌려 흐름 확인 (§3)
- [ ] **터미널 세팅**: 폰트 크게, 어두운 테마, 창 16:9 (§2)
- [ ] **녹화 시작**: ⇧⌘5 로 화면 기록 (§4)
- [ ] **실행**: `python scripts/demo.py` (실제 모델) (§4)
- [ ] **녹화 종료 → 편집**: 대기 구간 배속, 컷 정리 (§5)
- [ ] **마무리 컷**: 생성된 산출물·슬라이드 화면 (§6)

빠른 시작(준비가 끝났다면):
```bash
python scripts/demo.py        # 화면 녹화를 켠 상태에서 실행
```

## 1. 준비 (한 번만)

```bash
# 1) 최신 코드
git pull

# 2) 설치 (가상환경 권장)
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"        # 또는: bash scripts/setup.sh

# 3) 오프라인 검증 — 모델 없이 전 시나리오가 도는지
pytest -q                      # 통과하면 코드 정상

# 4) 실제 모델 준비 확인 (가장 중요)
which claude                   # claude CLI 가 PATH 에 있어야 함
claude --print --model claude-haiku-4-5 "OK 라고만 답해"
```
- 4)에서 응답이 오면 **구독 로그인 상태 = 실제 모델 준비 완료**.
- `claude` 가 없거나 로그인 안 돼 있으면 데모가 자동으로 **오프라인(예시) 모드**로 돈다(영상은 되지만 산출물 내용이 단순해짐). 실제 모델 영상을 원하면 먼저 Claude Code 로그인을 해둘 것.

## 2. 터미널 세팅 (영상 품질)

- **iTerm2**(권장) 또는 기본 **Terminal.app**.
- 폰트 **18~22pt**, 어두운 단색 테마(대비 좋게).
- 창 비율을 **16:9**에 가깝게, 탭·경로·알림 등 잡동사니 최소화.
- 작업 폴더는 프로젝트 루트(`archiagent/`).

## 3. 리허설 (녹화 전 1회)

```bash
python scripts/demo.py --fake      # 즉시·결정적, 모델 호출 0회
```
- 5.1 → 5.2 → 5.3 → 5.4 배너와 함께 흐름을 미리 본다. 실수 없이 갈 수 있는지 확인용.
- 타이핑이 빠르거나 느리면 `scripts/demo.py` 상단의 `TYPE`(글자 지연)·`PAUSE`(단계 간격)만 바꾼다.

## 4. 녹화 (본편)

1. **⇧⌘5** → "선택 부분 기록" → **터미널 창 영역** 선택 → **기록** 시작.
   - 고화질/유튜브용이면 **OBS Studio**(무료) 창 캡처가 더 깔끔하다.
2. 터미널로 돌아가 실행:
   ```bash
   python scripts/demo.py
   ```
3. 자동 재생된다:
   - 명령은 **타이핑 애니메이션**으로 입력됨
   - 모델이 생각하는 동안 **"⠙ 처리 중…"** 스피너가 돎(정지 화면 방지)
   - 저장 게이트는 미리보기 후 **자동 `y`**
   - 총 모델 호출 약 7회(각 10~15초) → 원본 길이 약 2~3분
4. "시연 종료." 가 뜨면 **⌘⌃Esc**(또는 메뉴바 ■)로 녹화 종료.

> 실제로 사람이 치는 라이브 느낌을 원하면 데모 플레이어 대신:
> ```bash
> archiagent examples/bank-nextgen
> ```
> 프롬프트에 §7의 명령을 직접 입력하고 게이트에 `y`.

## 9. 문제 해결

- **오프라인 모드로 뜬다("[오프라인(예시) 모드]")** → `claude` 미설치 또는 미로그인. §1-4로 확인.
- **너무 느리다** → 편집에서 대기 구간 배속. 또는 리허설·짧은 컷은 `--fake`로.
- **JSON 관련 오류** → 최신 코드인지 확인(`git pull`). 파서 이슈는 수정됨.
- **게이트에서 멈춘 것 같다** → 데모 플레이어는 자동 `y`. 라이브 CLI(`archiagent`)면 직접 `y` 입력.
- **다시 처음부터** → 저장은 세션 메모리라 프로그램을 다시 실행하면 초기화된다.

## 10. 조절 포인트

- 타이핑/멈춤 속도: `scripts/demo.py`의 `TYPE`·`PAUSE`.
- 시나리오 폴더 바꾸기: `python scripts/demo.py --dir <폴더>`.
- 필요하면 요청: `--slow`/`--fast` 옵션, 데모별 개별 실행, "무관 변경은 설계 안 건드림" 판단 컷 추가.
